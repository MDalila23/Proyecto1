# Proyecto de Registro de Pruebas de QA

## Descripción General
Este proyecto está diseñado para documentar y gestionar pruebas de aseguramiento de calidad (QA) realizadas sobre un sistema o aplicación. El objetivo principal es llevar un control detallado de los errores encontrados, los pasos para reproducirlos, los resultados esperados y actuales, así como la severidad de cada incidencia.

## Estructura del Archivo Excel
El archivo `proyecto_1.xlsx` contiene la información central del proyecto. Cada fila representa un caso de prueba o un error identificado durante el proceso de QA. La estructura de las columnas es la siguiente:

1. **ID del error:** Identificador único para cada error o incidencia.
2. **Título:** Breve descripción o nombre del error.
3. **Pasos a seguir:** Instrucciones detalladas para reproducir el error.
4. **Resultados esperados:** Comportamiento que debería observarse si el sistema funciona correctamente.
5. **Resultado actual:** Comportamiento real observado durante la prueba.
6. **Severidad:** Nivel de impacto del error (por ejemplo: Crítico, Alto, Medio, Bajo).

## Uso del Proyecto
1. Abre el archivo `proyecto_1.xlsx` para consultar o registrar nuevos errores.
2. Completa cada columna con la información correspondiente para mantener un historial claro y ordenado de las pruebas.
3. Utiliza la columna de severidad para priorizar la atención y resolución de los errores más críticos.

## Beneficios
- Facilita la trazabilidad de errores y pruebas.
- Permite priorizar incidencias según su severidad.
- Mejora la comunicación entre equipos de desarrollo y QA.
